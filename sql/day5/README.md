# Day 5 — DDL: CREATE TABLE, Constraints & Relationships

**Goal:** Define tables with primary keys, foreign keys, checks, and unique constraints; understand how schema enforces data integrity for ops automation.

**Time:** 4 hours

---

## 1. CREATE TABLE pattern

```sql
CREATE TABLE deployment_steps (
    id              BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    deployment_id   BIGINT NOT NULL REFERENCES deployments (id) ON DELETE CASCADE,
    step_name       TEXT NOT NULL,
    step_order      SMALLINT NOT NULL,
    status          TEXT NOT NULL CHECK (status IN ('pending', 'ok', 'failed')),
    started_at      TIMESTAMPTZ,
    finished_at     TIMESTAMPTZ,
    UNIQUE (deployment_id, step_order)
);
```

| Clause | Purpose |
|--------|---------|
| `PRIMARY KEY` | Unique row identity |
| `REFERENCES ...` | Foreign key to parent |
| `NOT NULL` | Column required |
| `UNIQUE` | No duplicate combinations |
| `CHECK (...)` | Domain validation |
| `DEFAULT` | Value when omitted on INSERT |

---

## 2. Identity columns vs SERIAL

Legacy:

```sql
id SERIAL PRIMARY KEY
```

Modern PostgreSQL (preferred):

```sql
id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY
```

`GENERATED BY DEFAULT` allows explicit ids on insert (migrations); `ALWAYS` requires `OVERRIDING SYSTEM VALUE` for seed scripts.

---

## 3. Foreign key actions

```sql
REFERENCES deployments (id)
    ON DELETE CASCADE    -- delete steps when deployment deleted
    ON DELETE RESTRICT   -- block delete if children exist
    ON DELETE SET NULL   -- orphan child FK column
```

Choose explicitly—default is `NO ACTION` (checked at end of statement).

## 4. ALTER TABLE (evolve schema)

```sql
ALTER TABLE services
ADD COLUMN slack_channel TEXT;

ALTER TABLE services
ADD CONSTRAINT services_slack_channel_chk
CHECK (slack_channel IS NULL OR slack_channel ~ '^#[-a-z0-9]+$');

ALTER TABLE hosts
ADD COLUMN retired_at TIMESTAMPTZ;
```

Production: run via migration tool (Day 14), not ad-hoc on prod.

---

## 5. DROP and TRUNCATE (lab only)

```sql
DROP TABLE IF EXISTS deployment_steps;
TRUNCATE deployment_steps RESTART IDENTITY CASCADE;
```

`TRUNCATE` is fast but bypasses row-level triggers—know your backup before using in prod.

## 6. Indexes at creation time


## 7. Comments for operators

```sql
COMMENT ON TABLE deployment_steps IS 'Ordered pipeline steps per deployment';
COMMENT ON COLUMN deployment_steps.step_order IS '1-based sequence; unique per deployment';
```

Visible in `\d+` and data catalogs—cheap documentation.

## 9. DevOps connections

- **IaC:** Terraform creates RDS; Flyway owns **inside** the database—both are required.
- **Zero-downtime:** Adding `NOT NULL` without default needs multi-phase migration (add nullable → backfill → enforce).
- **CI validation:** `sqlfluff lint` or `atlas schema diff` in pull requests.

## Quick reference

| Task | SQL |
|------|-----|
| Create table | `CREATE TABLE ... (...);` |
| PK | `id ... PRIMARY KEY` |
| FK | `col REFERENCES parent(id)` |
| Check | `CHECK (col IN (...))` |
| Add column | `ALTER TABLE t ADD COLUMN ...;` |
| Drop table | `DROP TABLE IF EXISTS t;` |
